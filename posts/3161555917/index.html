VMware 通过宿主机代理连接网络

发表于2024-07-21|更新于2024-07-21|经验记录
|总字数:1.7k|阅读时长:5分钟|浏览量:

因为测试和学习研究的需要，我使用了 VMware 软件创建与使用虚拟机。

上回安装完 Ubuntu 之后一直苦于代理工具配置不好，忽然想起可以让虚拟机通过本机代理上网……

现在部分开发学习环境可以放心挪到虚拟机里了。

本文所介绍仅供技术学习使用，不得利用本文内容进行违反法律、道德的行为，否则后果自负

准备工作

本文写作时的配置环境为：

  • 物理机 Windows 系统
  • VMware Workstation
  • V2-RayN（安装在物理机系统上）

为行文方便，本文可能使用到的例子如下：

  • 虚拟网络名称 -> VMnet8
  • 虚拟网络中主机的地址 -> 192.168.200.1

请先确保你即将使用的代理工具能够正常使用，例如正常访问 Google 搜索，否则建议先解决代理工具的问题。

知识铺垫

NAT 网络地址转换（Network Address Translation，简称 NAT）是用于在本地网络中使用私有地址，在连接互联网时转而使用全局 IP 地址的技术。NAT 实际上是为解决 IPv4 地址短缺而开发的技术，而且还能够有效地避免来自网络外部的攻击，隐藏并保护网络内部的计算机。

NAT 的实现方式有三种，即静态转换（Static Nat）、动态转换（Dynamic Nat）和端口多路复用（OverLoad）。

不过在这里，你只需要把它当做一个虚拟的特殊的路由器看待即可。VMware 虚拟网络功能可以虚拟出这样一个“路由器”，只需要让主机和虚拟机均连接至这个“路由器”，便可以进行通信。后文中若无其他说明，则提到的“路由器”均指这里的“虚拟网络”。

创建虚拟网络

此步骤没有必须修改之处，除非你不希望使用默认创建的虚拟网络。

VMware 默认会创建一个 NAT 类型虚拟网，如果没有或不确定其存在和名称，可进入 VMware 主界面，从上方菜单栏进入编辑->虚拟网络编辑器进行查看。通常情况下它会默认创建一个名为VMnet8的虚拟网，为方便起见，我们接下来便直接使用这个名为VMnet8的网。如果不是，则需要根据你的实际情况修改。

如果虚拟机的 NAT 模式出现问题，比如网卡不分配 IP、NAT 模式下无法上网，多半是 VMware 服务有问题，需要保证 VMware DHCP Service 和 VMware NAT Service 服务是正常运行的。如果不确定，可通过物理机Win+R->services.msc进入服务查看。

连接

还是上一步那个虚拟网络编辑器窗口，选中你准备连接的那个虚拟网络（例如本文中默认创建的VMnet8），将下方将主机虚拟适配器连接到此网络保持勾选状态，主机便已经连上该虚拟网络（即虚拟的“路由器”）。

之后选择你准备连接该“路由器”的虚拟机，点击编辑虚拟机设置，或右键左侧虚拟机列表中的虚拟机选择编辑，或点击上方菜单中虚拟机->设置，在弹出窗口中点击网络适配器，将网络连接一栏中选择自定义，选项框中选择前面你设置的那个网络（如果按照本文流程应为VMnet8（NAT 模式）。至此该虚拟机连上该“路由器”。

本机代理

打开主机的任务管理器，进入性能选项卡，找到前面你准备的网络（例如VMnet8，这里可能显示为以太网 VMware Network Adapter VMnet8），点击后在下方找到其IPv4 地址并记录（例如，192.168.200.1，这里需要根据自己实际情况进行调整，这是主机在这个“路由器”下的地址（其实在前面虚拟网络编辑器窗口中就已经能直接看到这个地址了，不过无妨）。

然后打开主机上的代理工具，这里以 V2RayN 为例。点击上方菜单栏设置->参数设置->Core: 基础设置，将允许来自局域网的连接开关打开，点击下方的确定。该软件默认端口如下：

请记下这里的端口号。如果在刚才设置界面对端口有更改则需根据你的情况更换端口（V2RayN 软件主界面下方可以找到这个端口）。

如果你使用的是其他代理软件，则需同样将其对应实现局域网连接的功能启用。

虚拟机代理

先在主机代理软件中配置一个速度尚可、尽量稳定的连接线路，在本机测试访问 Google、GitHub 等在国内通常情况下难以顺利访问的网站，若能够正常访问，则不必再切换线路，而是进行接下来的操作。

Windows10

进入 Windows 虚拟机的设置->网络和 Internet->代理，下拉找到手动设置代理，将使用代理服务器打开，在前面填入虚拟网络的 IPv4 地址（前面在任务管理器查看的那个地址，例如本文举例的192.168.200.1），后面填入 HTTP 代理端口（例如 V2-Ray 默认的10809，这里不填 SOCKS5 端口）。

然后在虚拟机里测试访问国外网站，若主机代理软件日志中出现虚拟机中网站访问记录，且能正常加载网站，则表明配置生效。

如果代理后无法上网，可以在命令提示符中ipconfig一下，确认当前连接的 IP 地址。

Ubuntu

启动系统，进入设置->网络，在页面中找到网络代理，点击右边的齿轮按钮并改为手动，填写代理地址（按照本文例子填写，请根据自身情况修改！）：

项目名称地址端口
HTTP 代理192.168.200.110809
HTTPS 代理192.168.200.110809
FTP 代理【此处不需要填写】【保持默认即可】
Socks 主机192.168.200.110808
忽略主机【保持默认即可】【无】

然后在虚拟机里测试访问国外网站，若主机代理软件日志中出现虚拟机中网站访问记录，且能正常加载网站，则表明配置生效。

后记

Linux 系统中一些程序（例如 wget、curl）等可以自动读取网络代理变量，可以自动检测并使用代理；有些工具可能还需手动配置。
