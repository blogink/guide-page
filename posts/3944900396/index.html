提示词反推工具 WD-Tagger 本地部署

发表于2025-02-02|更新于2025-02-02|经验记录
|总字数:2.4k|阅读时长:8分钟|浏览量:

在我的设备上，ComfyUI 中的 WD-Tagger 无法正常工作，但在 WebUI 中的可以。

按照网上的教程在 ComfyUI 中重装 onnxruntime 未果，继续找寻，发现了 HuggingFace 上的一个 演示站。与插件版相比有所不同，但基础的部分是一样的，都是上传图片，选择模型，分析提示词。

有时我只想简单地用 WD 标签器分析下图片的标签而不是立即绘图，不想仅仅为了这个目的而启动 WebUI（虽说反推图像提示词不一定要 WD 标签器）；在线运行受到网络条件限制，那就在本地部署一份。

环境配置

此节的 Python 和 Git 配置流程不是完整的配置流程，只能达到运行 wd-tagger 的目的。如非必要，建议走正常安装的流程，可以避免许多问题。如果你有全局 Python 环境和 Git，或是使用了环境隔离工具（例如 Anaconda），那么可以直接跳过此节。

另外，建议全程准备加速器，例如 Watt Toolkit（目前它还能加速 HuggingFace）。

路径以E:\wd-tagger为例。

Python

先配置 Python 环境，可以前往 Welcome to Python.org 选择与自己系统匹配的版本；也可以安装 Anaconda，更方便地管理 Python 环境。此二者配置较为简便，不在此赘述。如果你选择正常安装 Python 环境，则从上面两个链接选一并按照安装程序和文档进行配置，然后直接阅读本文下一节。

但有时考虑到便携、纯净性、Runtime 大小等因素，需要使用嵌入版的 Python（相对于“绿色版”）。前往 Python Releases for Windows 寻找适合的版本，并找到有Download Windows embeddable package字样的下载链接。这里我们选择 3.10.11 嵌入版作为演示（amd64 下载链接），将其下载并解压至本地，例如E:\wd-tagger\python。

Embeddable 版本主打的是便携性，和标准发行版相比精简了很多东西，但绝大部分库仍可在此环境下运行。仅为了运行 wd-tagger 则不需要特别配置环境变量、Tkinter 等，但是 pip 还是需要的。

前往 pip · PyPI 页面，点击Project description->Installation->跳转页面中get-pip.py一节，将 https://bootstrap.pypa.io/get-pip.py 文件保存至 Python 根目录（E:\wd-tagger\python\get-pip.py）。

Git

前往 Git 官网下载适合自己系统的版本并安装。Linux 上自带 git 而 Windows 不然。如果你选择正常安装 Git，则从官网下载安装包并按照安装程序和文档进行配置，然后直接阅读本文下一节。

如果你追求便携，同样的，git 也有便携版（git-portable）。在官网中下载64-bit Git for Windows Portable这个版本（自备 GitHub 加速工具），将其解压至本地，例如E:\wd-tagger\git。

（这里只记录了我使用的参数）逐行看下：

  1. 切换当前工作目录到E:\wd-tagger，后续的命令将在这个目录下执行；
  2. 由于国内网络的原因，无法直连 HuggingFace，因此我们需要手动设置地址，如 hf-mirror 镜像站。将环境变量 HF_ENDPOINT 的值设置为 https://hf-mirror.com，用于指定 Hugging Face 的镜像地址；
  3. Gradio 默认监听端口为 7860，为防止与其他同样使用此端口的程序冲突，可以设置 GRADIO_SERVER_PORT 环境变量，将 Gradio 的监听的端口设置为其他值，例如7861；
  4. 在Windows 系统中，HuggingFace 默认储存路径为C:\Users\【用户名】\.cache\huggingface\hub。为方便管理，此处可以设置 HF_HOME 环境变量，指定 Hugging Face 的存储路径为E:\wd-tagger\hf_home（需要自己创建这个路径）；
  5. 执行app.py，使用E:\wd-tagger\python\python.exe作为 Python 解释器，启动 webui 界面 。

中间那些命令是设置自定义参数的，非启动程序所必需。按照这个脚本，启动后需在浏览器输入127.0.0.1:7861以打开 webui 界面。

使用

此时本地还没有推理模型。在 webui 中选择模型后，首次使用会连接 HuggingFace 下载选中的模型，之后使用都从本地读取。如果按照前一节那个示例脚本启动程序，那么这些模型会被下载到前面设置的hf_home目录内对应的文件夹中。

这一步对网络要求较高，而且模型文件较大；如果下载失败，可以多试几次。嫌速度慢的，可以手动下载，然后放入 HF 缓存文件夹相应目录中。为了方便说明，此处以 wd-vit-tagger-v3 模型为例。先前往 SmilingWolf（作者主页，此处为镜像站链接）选择适合的模型，下载model.onnx和selected_tags.csv备用。

在 webui 中选择模型并点击Submit按钮，当E:\wd-tagger\hf_home\hub中创建相应文件夹后关闭程序（文件夹命名形式类似于models--SmilingWolf--wd-vit-tagger-v3），其中结构大致为：

该文件夹中 snapshots 目录内有名称为字母数字混合的文件夹（大致是校验码和软链接，不理解也不影响使用），对于本文选用的反推工具而言，完整下载过程结束后，相应的model.onnx和selected_tags.csv会出现在这个目录内。此时只需将snapshots中字符文件夹内的文件删除，将前面提前下载好的model.onnx和selected_tags.csv移动至此，重启程序脚本，选中即可正常使用该模型。
