VMware 虚拟机安装教程

发表于2024-07-17|更新于2024-07-17|经验记录
|总字数:2.3k|阅读时长:7分钟|浏览量:

（本想记录一个 VMware 提示驱动问题的解决过程，但仅写这个会导致文章过于短，于是顺便吧 VMware 的安装流程记录以下。）

简要介绍

VMware Workstation Pro（后文为方便将其简称为 VMware）是业界标准的桌面 Hypervisor，用于在 Linux 或 Windows PC 上运行虚拟机。它允许您在单一的桌面上同时运行不同的操作系统，是开发、测试和部署新应用程序的最佳解决方案。

准备工作

  • 邮箱（最好是 Outlook、Gmail 之类的，尽量不要国内邮箱。如果你的 VMware 不是在官网下载的那么可以忽略这条）
  • 操作系统镜像（尽量准备 ISO 格式的）
  • 懂得如何使用搜索引擎

下载

2023 年 11 月 23 日，博通正式宣布完成对虚拟软件服务商 VMware 的收购，交易规模约为 690 亿美元，其中包括 610 亿美元的协约收购价和后者 80 亿美元的债务。2024 年 5 月 14 日，VMware 发布消息称，Workstation Pro 现免费面向个人使用（自版本 17.5.2 开始）。

由于调整，VMware 软件下载链接已经合并到 Broadcom 网站，原有网站下载已停用，包括旧版 VMware 更新检测可能也无法使用。如果仍然按照原先的方式去下载，可能不成功。

现在下载需要前往 www.broadcom.com 注册账户并登录（国内部分邮箱注册会提示失败，建议使用outlook、gmail等）。之后点击顶部导航栏的Products->VMware Products，在下方Results处选择T-Z，下拉找到Workstation Pro，点击跳至VMware Desktop Hypervisors页面（嫌点击麻烦的可以在登录后访问 此处 直达），找到DOWNLOAD FUSION OR WORKSTATION按钮，点击进入My Downloads - VMware Cloud Foundation，拉到底下点击VMware Workstation Pro即可进入相关下载页面。

有点麻烦，而且 VMware 17.5.2 版本的新的下载地址也有点问题，但 17.5.0 及以前的版本基本可以正常下载。目前可以临时通过网上有人分享出来的第三方下载地址。这里提供我使用的下载地址：CDS Repository 。

Hyper-V 相关

有些时候 Hyper-V 会跟大部分模拟器、VMware 这些虚拟化软件冲突。启用 Hyper-V 后，Windows 应用层相当于成为了一个运行在 Hyper-V 上的虚拟机，然而 VMware 不能在虚拟机环境中运行，于是乎……不过自 VMware 15.5.5 起，这个问题得到了改善。你只需要在安装完的虚拟机中右键点击设置->硬件找到处理器一项，将“虚拟化引擎”中几项均打钩，即可实现二者共存。

如果是家庭版，一般是没有这个功能的（其实你也可以按照网上一些操作去启用这个功能）。

你也可以通过Win+R（运行）->输入msinfo32进入“系统信息”窗口，在系统摘要一项拉到最下面，如果四个“Hyper-V”项目的值都为是，则表明该设备可以运行 Hyper-V 虚拟机。

在控制面板->程序->启用或关闭 Windows 功能中有Hyper-V一项。如果想关闭 Hyper-V 功能，需将此处取消框选，然后重启系统。

总而言之，家庭版用户不需要管这里，跳过此步即可。

安装

双击安装包，出现“欢使用 VMware Workstation Pro 安装向导”界面，点击下一步。在“最终用户许可协议”中勾选“我接受许可协议中的条款”，再次点击下一步。

如果你已经安装过 VMware，则接下来的页面是“更改、修复或删除安装”，可根据自己需求使用其提供的三个处理方法。

如果没有安装过，则接下来的页面应为“自定义安装”。在这个界面你可以更改一下安装位置（改完后程序如果没有自动创建 VMware 文件夹，记得手动创建一个）。下面“增强型键盘驱动程序”和“将 VMware Workstation 控制台工具添加到系统 PATH”两项可根据自己需求勾选（不确定的也建议都勾选下），之后点击下一步。

在“用户体验设置”中，将“启动时检查产品更新”和“加入 VMware 客户体验提升计划”此两项勾选去掉（其实勾选也没有问题，只是某些情况下自动更新弹窗会让人觉得不爽），之后点击下一步。

在“VMware Workstation 安装向导已完成”中，你可以选择许可证或完成退出向导。许可证可以去网上找，不过那种一般是允许个人使用的，商用不建议；可能输完许可证不能通过，那就多试几个，总有能过的。（按照博通的消息，未来这个界面将会有个人使用的选项，个人用户可能不需要填写许可证就能完整使用）

完成后按照窗口提示重启系统即可完成整个安装流程。

安装虚拟机

创建虚拟磁盘

在 VMware 软件左上角进入文件->新建虚拟机，打开“新建虚拟机向导”窗口。通常情况下选择第一项（典型）即可满足大部分需求，然后点击下一步。

进入“安装客户机操作系统”，选则稍后安装操作系统一项（如果选择安装程序光盘映像文件也行，不过有些镜像会出现该操作系统将使用简易安装，为减少一些奇怪问题，不如不选此项），之后点击下一步。

“选择客户机操作系统”，不必言说，按照你准备安装的虚拟机的镜像进行选择，然后点击下一步。

“命名虚拟机”，选择虚拟机位置和名称。最好不要设置与已安装的其他虚拟机重复（如果是安装你这个设备上第一个 VMware 虚拟机则不用管）；虚拟机位置建议挪到别的盘中，如果你硬盘足够大那当我没说。完成后点击下一步。

“指定磁盘容量”，它默认给出的容量一般是考虑很多方面因素的。极限情况下硬盘可以不需要这么大，例如 Windows10 某些镜像你调 20GB 也不是不能用（MacOS 另当别论，装过就知道了）。还是一样，主机硬盘够大就按你心情设置即可。下面两个选项是虚拟磁盘的储存方式，按照自己需求选择就行，我选择的是将虚拟磁盘拆分成多个文件。完成后

“己准各好创建虚拟机”，如果你还有其他需求，可以点击自定义硬件进行调整，完成后点完成即可创建虚拟磁盘。刚才的硬件设置，可以在库这一栏找到你的虚拟机，选中后右键点击设置进行编辑。

挂载镜像与安装系统

在库这一栏找到你刚才创建的虚拟机，选中后右键点击设置->硬件选项卡，找到CD/DVD (SATA)一项，选中使用 ISO 映像文件，点击浏览，找到你准备安装的操作系统的 ISO 镜像文件，点击确定。

选中该虚拟机，在右侧找到开启此虚拟机。如果上一步的镜像文件正常，则会自动进入安装界面。

操作系统的安装与使用，与在物理机上的安装方法相同，具体请自行使用搜索引擎了解，这里无法一一列出。

后续

与 vmx86 驱动程序的版本不匹配

在安装完该软件之后，会在C:\windows\system32\drivers目录下生成一个vmx86.sys驱动文件。如果在卸载、重装 VMware 时因操作或其他某些原因导致失误，可能会报这个错误。归根结底就是如报错所言：驱动程序“vmx86.sys”的版本不正确。

在这里有两个解决方法：

  1. 进入C:\windows\system32\drivers目录，找到vmx86.sys并将其重命名（随便什么名称都行），然后再次双击最开始你下载下来的 VMware 安装包（注意那个安装包的版本与你当前安装的 VMware 版本相同），选择“修复”，之后重启系统；
  2. 使用 VMware 安装包自带的卸载功能（双击安装包），将其完全卸载后重新安装（注意点不要一并清理了虚拟机磁盘所在的文件夹），之后重启系统。

MacOS 开机五国、无限重启
在安装或更新完VMware后，MacOS锁会重新锁上，需要通过unlocker删除后重新安装补丁。首次使用可以去paolo-projects/unlocker下载最新版本解锁工具，解压出来的文件夹放在与 VMware 同级目录下。

Windows 用户使用管理员权限运行win-build.cmd，等待下载与配置补丁；如果之前运行过此补丁程序，还需先用管理员权限运行win-uninstall.cmd卸载旧的补丁，再执行安装。
